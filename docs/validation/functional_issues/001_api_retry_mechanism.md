# Issue #1: API 客户端缺少请求重试机制

## 严重程度
**高** - 影响可靠性和用户体验

## 问题描述
API 客户端的 `RETRY_STATUS_CODES` 为空列表，导致网络故障或临时服务器错误时无法自动重试请求。这会降低 SDK 的可靠性，用户需要手动处理临时性错误。

## 当前状态
```python
# pyromind_sdk/client/base.py:21
RETRY_STATUS_CODES = []  # 空列表，不会重试任何状态码
```

## 影响
- 网络波动时请求直接失败
- 临时性服务器错误（503、504）无法自动恢复
- 用户体验差，需要自己实现重试逻辑
- 在不稳定网络环境下几乎无法使用

## 预期行为
应该对临时性错误自动重试：
```python
RETRY_STATUS_CODES = [408, 429, 500, 502, 503, 504]
```

## 建议修复
1. 添加默认重试状态码列表
2. 允许用户自定义重试策略
3. 添加指数退避算法
4. 记录重试日志便于调试

## 功能优先级
**高** - 这直接影响 SDK 在生产环境的可用性
